CENTRAL EXECUTION STANDARD

- FAQ tracking

- correct body data-page-type, data-service, and data-primary-goal

GLOBAL HEADER / FOOTER STANDARD
Every page must include these placeholders:

<div id="siteHeader"></div>
<div id="siteFooter"></div>

Every page must load the global script once:

<script src="/assets/js/site.js"></script>

Rules:
- Do not hardcode the header or footer inside individual pages.
- Do not duplicate header/footer markup per page.
- Do not add page-specific header/footer injection scripts if /assets/js/site.js already loads includes.
- The current site.js already loads /assets/includes/header.html and /assets/includes/footer.html through loadIncludes().
- Do not add a second include loader.
- Do not include /assets/js/site.js more than once.
- Header and footer links should remain tracked through their existing data-track-* attributes.

GLOBAL CSS STANDARD
Every page must use the global stylesheet:

<link rel="stylesheet" href="/assets/css/site.css">

Rules:
- Use existing global classes from site.css whenever possible.
- Do not add page-level CSS unless absolutely necessary.
- Do not restyle header or footer in page files.
- Do not create one-off layout systems when current global classes can handle the page.
- If page-level CSS remains, explain why in the QA summary.
- Optimize for both mobile and desktop using global patterns already present in site.css.
